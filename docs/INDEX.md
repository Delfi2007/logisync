## üìù Documentation Standards

### File Naming Convention
- Use descriptive, ALL_CAPS names with underscores
- Include version dates when relevant
- Prefix with category (e.g., `PHASE4_`, `SESSION_`)

### Organization Principles
1. **Getting Started** - Everything needed to start
2. **Development** - Everything for active development
3. **Deployment** - Everything for deploying
4. **Phase Completion** - Historical records of completed work
5. **Project Management** - Planning and tracking
6. **Reference** - Quick lookups

### Document Structure
- Always include: Title, date, status
- Use clear headings and table of contents
- Include code examples where relevant
- Link to related documentation
